Git 命令操作笔记

安装

下载Git - git 官网. - 淘宝镜像 Git for Windows 安装步骤略

配置

对你的commit操作设置关联的用户名

对你的commit操作设置关联的邮箱地址 查看您的配置 启用有帮助的彩色命令行输出

创建仓库

创建一个新的本地仓库

将本地仓库与一个 GitHub 上的空仓库连接起来 Clone（下载）一个已存在于 GitHub 上的仓库，包括所有的文件、分支和提交(commits)

.gitignore 文件

有时一些文件最好不要用 Git 跟踪。这通常在名为 .gitignore 的特殊文件中完成。你可以在 github.com/github/gitignore 找到有用的 .gitignore 文件模板。

进行更改

浏览并检查项目文件的发展

列出当前分支的版本历史 列出文件的版本历史，包括重命名 展示两个分支之间的内容差异 将文件进行快照处理用于版本控制（暂存） 将文件快照永久地记录在版本历史中（提交）

重做提交（不推荐）

清除错误和构建用于替换的历史 撤销所有 [commit] 后的的提交，在本地保存更改

放弃所有历史，改回指定提交。

分支

创建一个新分支

切换到指定分支并更新工作目录(working directory) 将指定分支的历史合并到当前分支。这通常在拉取请求(PR)中完成，但也是一个重要的 Git 操作。 删除指定分支

同步更改

下载远端跟踪分支的所有历史（获取）

将远端跟踪分支合并到当前本地分支 将所有本地分支提交上传到 GitHub 使用来自 GitHub 的对应远端分支的所有新提交更新你当前的本地工作分支。（拉取） git pull 是 git fetch 和 git merge 的结合
