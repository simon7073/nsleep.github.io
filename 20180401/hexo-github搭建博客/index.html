CRLF问题(折叠)
首先问题出在不同操作系统所使用的换行符是不一样的，下面罗列一下三大主流操作系统的换行符： Uinx/Linux采用换行符LF表示下一行（LF：LineFeed，中文意思是换行）； Dos和Windows采用回车+换行CRLF表示下一行（CRLF：CarriageReturn LineFeed，中文意思是回车换行）； Mac OS采用回车CR表示下一行（CR：CarriageReturn，中文意思是回车）。 为true时，Git会将你add的所有文件视为文本问价你，将结尾的CRLF转换为LF，而checkout时会再将文件的LF格式转为CRLF格式。 为false时，line endings不做任何改变，文本文件保持其原来的样子。 为input时，add时Git会把CRLF转换为LF，而check时仍旧为LF，所以Windows操作系统不建议设置此值。

Js.org

使用Js.org和自定义域名类似，将一个名为“CNAME”的文件添加到您的存储库中，然后在GitHub repository 中发出拉取请求，该请求会将子域添加到现有JS.ORG域的列表中。

第一步，fork 仓库。

进入 js.org 的主仓库，点击右上角的 Fork 按钮。稍等一下刷新页面。这样，这个仓库就被 fork 到你自己的名下。

第二步，修改代码。

将 fork 过来到仓库 clone 到本地，然后用编辑器打开，找到 cnames_active.js 文件，在里面看一下，大概知道怎么回事了吧。添加你想要到二级域名，以及 CNAME 到的网址。注意，大家的约定都是按字母顺序排序的。照葫芦画瓢改好代码。push 回 github。

  • 比如："isimon": "nsleep.github.io",
  • CNAME文件：isimon.js.org

第三步，Pull Request。

回到你 fork 的仓库页面，找到一个叫 Pull Request 的按钮。点击它，向js.org发送一个申请
内容如下：
标题：你要申请的域名，如：isimon.js.org
内容：（阅读协议，不用更改）
点击 Create pull request 提交。

这样就完成了域名申请和绑定了。是不是很简单。接下来的事情，就是等待，因为中美之间的时差关系，你最好在晚上 12 点之前完成上面这些操作（根据indus的提醒，他实际上是在德国工作，因此时间比我们早）。等美国那边的项目维护者上线之后，就会对你的 pull request 进行检查，如果没有问题，就会 approve，一切顺利，就会 merge，之后很快你就可以用拿到的域名访问你的 gh-pages 了。

如果你有兴趣，还可以继续看看 ns_active.js，看完之后，你就能再次明白，js.org 就是一个域名服务项目啊。
